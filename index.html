Web Uygulamanızı GitHub Pages'te Görüntüleme

Neden sadece "hizliokuma.github.io" görüyorsunuz ve nasıl düzeltirsiniz?

Sorun: Sadece "hizliokuma.github.io" Görüyorum

GitHub Pages başarıyla kuruldu ancak web sitenizde sadece "hizliokuma.github.io" yazıyorsa, bunun birkaç olası nedeni vardır:

Olası Nedenler:

  • Repository'nize henüz hiç dosya yüklemediniz
  • Yüklediğiniz dosyalar doğru konumda değil
  • Ana sayfa dosyanızın adı index.html değil
  • Dosyaları yüklediğiniz branch doğru değil

Çözüm: Dosyalarınızı GitHub'a Yükleyin

1

Repository'nize Dosya Yükleyin

GitHub'da repository'nizin ana sayfasına gidin ve "Add file" > "Upload files" butonuna tıklayın.

Web uygulamanızın tüm dosyalarını sürükleyip bırakın veya "choose your files" ile seçin.

Önemli: Ana sayfa dosyanızın adı mutlaka index.html olmalıdır.

2

Doğru Dosya Yapısını Kontrol Edin

Repository'nizdeki dosya yapısı şöyle görünmelidir:

hizliokuma.github.io/
  index.html     (Ana sayfa - zorunlu)
  styles.css     (Stil dosyası - isteğe bağlı)
  script.js     (JavaScript dosyası - isteğe bağlı)
  images/     (Resimler klasörü - isteğe bağlı)
    logo.png
3

Değişiklikleri Kaydedin

Dosyaları yükledikten sonra, değişiklikleri açıklayan bir mesaj yazın (örneğin: "İlk web uygulamamı yükledim") ve "Commit changes" butonuna tıklayın.

4

GitHub Pages Ayarlarını Kontrol Edin

Repository ayarlarında (Settings > Pages) aşağıdaki ayarların doğru olduğundan emin olun:

  • Source: Deploy from a branch
  • Branch: main (veya master) ve /root

Son Kontroller

1

Sayfayı Yenileyin

Dosyaları yükledikten sonra, hizliokuma.github.io adresini yenileyin. Değişikliklerin görünmesi birkaç dakika sürebilir.

2

GitHub Pages Durumunu Kontrol Edin

Repository'nizin Settings > Pages bölümüne giderek yayınlama durumunu kontrol edin. Yeşil bir kutucuk görüyorsanız, siteniz başarıyla yayınlanmış demektir.

3

Tarayıcı Önbelleğini Temizleyin

Bazen tarayıcı önbelleği eski sürümü gösterebilir. Ctrl+F5 tuşlarına basarak sayfayı tamamen yenileyin.
